# Release adapter templates

Project-owned adapters implement two commands:

```text
<adapterCommand> plan --context <context.json> --output <plan.json>
<adapterCommand> build --context <context.json> --output-dir <directory> --manifest <assets.json>
```

All JSON documents use `contractVersion: 1`. Unknown properties are rejected.

Steward invokes each phase independently with `shell: false`; `adapterCommand` is an argument array, not a shell expression. The adapter runs from the checked-out consumer repository, while all protocol files and build outputs live in a fresh runner-temporary directory. After `plan`, Steward checks existing tag/Release state before deciding whether `build` is needed. Steward inventories and hashes the output directory after `build` before accepting the assets manifest.

Manifest `release.triggerPaths` use the same case-insensitive repository-path matching as Steward Classification: `*` matches within one path segment and `**` may cross path separators.

The context is generated by Steward and binds the repository, merged pull request, and target commit:

```json
{
  "contractVersion": 1,
  "repository": { "id": 1296725030, "fullName": "splrad/steward-sandbox" },
  "pullRequest": { "number": 14, "mergeSha": "0123456789012345678901234567890123456789" }
}
```

`plan` writes project version facts only:

```json
{
  "contractVersion": 1,
  "displayVersion": "1.2.3",
  "buildId": "1.2.3+0123456",
  "tagName": "v1.2.3",
  "releaseTitle": "Example v1.2.3"
}
```

`build` writes assets inside `--output-dir` and a manifest such as:

```json
{
  "contractVersion": 1,
  "assets": [
    {
      "path": "example-v1.2.3.zip",
      "name": "example-v1.2.3.zip",
      "mediaType": "application/zip",
      "size": 42,
      "sha256": "0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef"
    }
  ]
}
```

Paths are POSIX-style and relative to the output directory. Steward rejects absolute or traversing paths, backslashes, missing/empty/non-file assets, case-insensitive duplicate paths or upload names, and declared size or SHA-256 values that do not match the generated file.

Adapters must not validate GitHub events, create tags or Releases, generate shared notes, upload assets, or write final Checks. Those remain Steward responsibilities.
